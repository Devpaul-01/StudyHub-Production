Why does quicksort have O(n²) worst case?
🧠 Learnora — Deep Think
Great question! The O(n²) worst case occurs when we consistently pick the smallest or largest element as pivot. Here's why...
Think of it like sorting a deck of cards: if you always pick the first card as your pivot and the deck is already sorted, you never split it efficiently — one side always has 0 cards, and the other has n-1. That's n recursive calls each doing n work = O(n²).
